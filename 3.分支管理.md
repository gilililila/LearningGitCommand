当我们在GitHub中与他人合作完成一个项目的时候，我们不会直接修改主分支，我们所有的修改将在其他分支中进行，确认无误后才合并到主分支中。

- **Master分支** 是已经确定的分支。
- **Feature分支** 相当于草稿纸，可以随意修改，不会影响到主分支。

### 查看分支
首先确定自己的分支在哪。在终端处输入
```Bash
git branch
```
我们应该会看到`* master`，星号表示当前所在的位置。

### 创建并切换到“草稿”分支
假设我们现在需要给笔记仓库添加一个`README.md`的介绍文件。我们需要创建一个`feature/add-readme`分支。
使用命令：
```Bash
git switch -c feature/add-readme
```
- `-c`意为创建
- `feature/add-readme`是社区通用的命名规范，表示这是一个开发新功能的分支。
现在再次输入`git branch`，星号应该在`feature/add-readme`前。

### 在“草稿”分支中操作
1. 创建一个`README.md`文件。随便写点东西。
2. 按照常规操作提交更改。
